# 系统部署指南

## 1. 系统环境要求

### 1.1 后端环境
- Python 3.6+
- Flask
- Flask-SQLAlchemy
- SQLite (默认) 或 SQL Server

### 1.2 前端环境
- Node.js 14+
- npm 6+
- Vue 3
- Vuex 4
- Vue Router 4
- Element Plus
- Chart.js
- Frappe Gantt

## 2. 后端部署

### 2.1 安装依赖
1. 克隆项目代码
   ```bash
   git clone <repository-url>
   cd TodoList
   ```

2. 创建虚拟环境
   ```bash
   python -m venv venv
   ```

3. 激活虚拟环境
   - Windows:
     ```bash
     venv\Scripts\activate
     ```
   - Linux/Mac:
     ```bash
     source venv/bin/activate
     ```

4. 安装依赖包
   ```bash
   pip install Flask Flask-SQLAlchemy
   ```

### 2.2 数据库配置
1. 编辑 `config.py` 文件，修改数据库连接字符串
   - 使用 SQLite (默认):
     ```python
     SQLALCHEMY_DATABASE_URI = 'sqlite:///todo.db'
     ```
   - 使用 SQL Server:
     ```python
     SQLALCHEMY_DATABASE_URI = 'mssql+pyodbc://username:password@server/database?driver=ODBC+Driver+17+for+SQL+Server'
     ```

2. 初始化数据库
   ```bash
   python init_db.py
   ```

### 2.3 启动后端服务
```bash
python app.py
```

后端服务将在 `http://127.0.0.1:5000` 运行。

## 3. 前端部署

### 3.1 安装依赖
1. 进入前端目录
   ```bash
   cd frontend
   ```

2. 安装依赖包
   ```bash
   npm install
   ```

### 3.2 配置前端
1. 编辑 `frontend/src/store/index.js` 文件，修改 API 基础 URL（如果需要）

### 3.3 开发环境启动
```bash
npm run dev
```

前端服务将在 `http://localhost:5173` 运行。

### 3.4 生产环境构建
```bash
npm run build
```

构建后的文件将生成在 `frontend/dist` 目录中。

## 4. 生产环境部署

### 4.1 后端部署
1. 使用 Gunicorn 作为 WSGI 服务器
   ```bash
   pip install gunicorn
   gunicorn app:app -w 4 -b 0.0.0.0:5000
   ```

2. 配置 Nginx 作为反向代理
   ```nginx
   server {
       listen 80;
       server_name example.com;

       location /api {
           proxy_pass http://127.0.0.1:5000;
           proxy_set_header Host $host;
           proxy_set_header X-Real-IP $remote_addr;
           proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
           proxy_set_header X-Forwarded-Proto $scheme;
       }

       location / {
           root /path/to/frontend/dist;
           index index.html;
           try_files $uri $uri/ /index.html;
       }
   }
   ```

### 4.2 前端部署
1. 将构建后的 `frontend/dist` 目录复制到 Nginx 的 web 根目录

2. 配置 Nginx 静态文件服务（如上所述）

## 6. 常见问题和解决方案

### 6.1 数据库连接失败
- 检查数据库连接字符串是否正确
- 确保数据库服务正在运行
- 检查数据库用户权限

### 6.2 前端无法访问后端 API
- 检查后端服务是否正在运行
- 检查 CORS 配置
- 检查前端 API 基础 URL 是否正确

### 6.3 任务依赖关系创建失败
- 检查任务是否存在
- 检查是否已经存在相同的依赖关系
- 检查数据库表结构是否正确

### 6.4 甘特图不显示
- 检查任务是否设置了开始时间和结束时间
- 检查前端依赖是否正确安装
- 检查浏览器控制台是否有错误信息

### 6.5 系统性能问题
- 优化数据库查询
- 增加服务器资源
- 启用缓存机制

## 7. 系统维护

### 7.1 数据库备份
- 定期备份数据库文件
- 对于 SQL Server，使用 SQL Server 备份工具

### 7.2 日志管理
- 配置 Flask 日志
- 定期清理日志文件

### 7.3 系统更新
- 备份现有代码和数据库
- 拉取最新代码
- 重新安装依赖
- 重启服务

## 8. 安全配置

### 8.1 后端安全
- 设置强密码
- 启用 HTTPS
- 配置 CORS
- 定期更新依赖包

### 8.2 前端安全
- 防止 XSS 攻击
- 防止 CSRF 攻击
- 敏感信息加密

## 9. 监控和告警

### 9.1 系统监控
- 使用监控工具（如 Prometheus、Grafana）
- 监控服务器资源使用情况
- 监控 API 响应时间

### 9.2 告警配置
- 配置邮件告警
- 配置短信告警
- 配置微信告警

## 10. 故障排查

### 10.1 查看日志
- 后端日志：Flask 日志文件
- 前端日志：浏览器控制台
- Nginx 日志：Nginx 错误日志

### 10.2 常见错误及解决方案
- 500 错误：检查后端代码是否有错误
- 404 错误：检查 API 端点是否正确
- 401 错误：检查用户权限
- 403 错误：检查访问权限
